# 空状態メッセージの改善（レシピノート / 買い物リスト）

## 目的・背景

タブ別の空状態メッセージが、ユーザーに「次に何をすればここに何が出るのか」を
うまく伝えていない。

- レシピノート: 現状は `レシピを生成すると自動で保存されます`
  （`mobile/app/(tabs)/recipes.tsx:70`）
  - "生成" がアプリ内のどの操作を指すのか初見では分かりづらい
  - レシピノート単体で「生成」操作はなく、実際は **買い物リストで料理を追加 →
    AI 具材生成 / レシピ表示の流れの中で auto-save される** ため、誘導先は
    買い物リストである
- 買い物リスト: 現状は `リストは空です。料理や食材を追加しましょう`
  （`mobile/app/(tabs)/index.tsx:320`）
  - メッセージ自体はあるが、`料理` ボタンと `食材` ボタンの 2 アクションを
    持っている UI に対して文言がやや淡白。両アクションを示唆する形に揃えたい

レシピノート改名（`docs/plans/archive/` 行きの予定）と同じ「ユーザー目線の
言葉に揃える」流れの一環として、両タブの空状態を見直す。

## 対応方針

文言だけ変える小さな変更。レイアウトやスタイル、ロジック、ストアには触らない。
具体文言は本プランで案を提示し、実装時にユーザー確認を取って確定する。

### 文言（確定）

- レシピノート（検索なし時）: `買い物リストで料理レシピを検索すると表示されます`
  - 検索結果なしの分岐 (`search ? '検索結果なし' : ...`) は据え置き
- 買い物リスト: 現状維持 (`リストは空です。料理や食材を追加しましょう`)
  - 実機確認の結果、現状文言で問題ないと判断

## 影響範囲

### 変える
- `mobile/app/(tabs)/recipes.tsx:70` — 空状態メッセージ

### 触らない（当初プランから縮小）
- `mobile/app/(tabs)/index.tsx:320` — 買い物リストの空状態メッセージは現状維持

### 触らない（共通）
- スタイル (`emptyText`)、レイアウト、`FlatList` / `ScrollView` 構造
- ストア (`recipe-store`, `shopping-store`)、API クライアント
- テスト — `__tests__/` 配下に該当文言を期待するテストは無し（実装時に grep 確認）
- 検索結果 0 件の文言 (`検索結果なし`)
