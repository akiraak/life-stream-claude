# AI 機能アイデア一覧

## 概要

Life Stream アプリにおける AI（Claude）活用機能のアイデア集。
既存のレシピ推薦機能に加え、日常生活を支援する AI 機能を検討する。

---

## 1. スマート買い物提案

**概要**: 過去の買い物履歴を分析し、買い忘れしそうなアイテムを提案する。

- 購入頻度・周期を学習（例: 牛乳は週1回）
- 前回購入から一定期間経過したら通知
- 「そろそろ必要では？」とリストへの追加を提案

**必要な変更**:
- 購入履歴テーブルの追加（チェック済みアイテムの保存）
- 周期分析ロジック
- プッシュ通知（iOS）

---

## 2. カテゴリ自動分類

**概要**: アイテム追加時に AI がカテゴリを自動判定する。

- 「牛乳」→ 乳製品、「鶏もも肉」→ 肉類、「トイレットペーパー」→ 日用品
- 既存の `category` フィールドを活用
- カテゴリ別のグループ表示でリストを見やすく

**必要な変更**:
- アイテム追加時に Claude API でカテゴリ推定
- Web/iOS でカテゴリ別グループ表示

---

## 3. 献立プランナー（1週間分）

**概要**: 買い物リストの食材から1週間分の献立を自動生成する。

- 栄養バランスを考慮した献立提案
- 曜日ごとの朝・昼・夜メニュー
- 足りない食材をリストに自動追加
- 既存のレシピ推薦機能を拡張

**必要な変更**:
- 献立テーブルの追加
- 献立表示用の Web ページ
- レシピ推薦 API の拡張

---

## 4. 予算管理アシスタント

**概要**: 買い物リストの合計金額を推定し、予算内に収める提案をする。

- 一般的な価格データをもとに概算を算出
- 予算オーバー時に代替品や節約提案
- 月間の買い物傾向レポート

**必要な変更**:
- 価格フィールドの追加（手動入力 or AI 推定）
- 予算設定機能
- レポート表示ページ

---

## 5. 自然言語でのアイテム一括追加

**概要**: 自由文から買い物アイテムを抽出してリストに追加する。

- 入力例: 「今夜カレーを作りたい」→ じゃがいも、にんじん、玉ねぎ、カレールー、肉を追加
- 入力例: 「牛乳と卵、あとパンも」→ 3アイテムを一括追加
- 音声入力との相性が良い

**必要な変更**:
- 自然言語解析用の Claude プロンプト
- 一括追加 API
- Web/iOS の入力 UI 拡張

---

## 5.5. 料理追加時の AI 具材・レシピ提案 ✅

**概要**: 料理をリストに追加する際、Gemini AI がその料理に必要な具材とレシピ3件を自動で調べて表示する。

- 料理追加 → バックグラウンドで AI が具材+レシピを取得
- 具材はトグルチップで表示（タップで選択/解除）
- レシピ3件をカード形式で表示（タイトル・概要・折りたたみ手順）
- AI 情報は dishes テーブルに永続化、同名料理の再追加時に引き継ぎ
- 同じ料理内の同名アイテム重複を防止

**実装済み**:
- Gemini API（`gemini-2.5-flash`）で具材+レシピ取得
- `POST /api/dishes/:id/suggest-ingredients` API（DBキャッシュ付き）
- Web: トグルチップ+レシピカードモーダル
- dishes テーブルに `ingredients_json`/`recipes_json` カラム追加

## 7. 店舗別ルート最適化

**概要**: 買い物リストのアイテムを店舗の売り場順に並べ替える。

- 複数店舗の場合、どの店で何を買うか振り分け

**必要な変更**:
- 店舗情報・売り場カテゴリの管理
- 並べ替えロジック

## 8. 季節・イベント提案

**必要な変更**:

## 96. 文脈理解メモ

## 99. 災害時モード

**概要**: 災害発生時に備蓄リストの確認、避難所情報、必要物資の優先順位を AI が提示する。

## 100. ライフログ統合ダッシュボード

## 優先度の目安

| 優先度 | アイデア | 理由 |
|--------|----------|------|
| 高 | 5.5. 料理追加時の AI 具材提案 | 既存の料理機能の自然な拡張、UX 向上大 |
| 高 | 2. カテゴリ自動分類 | 既存フィールド活用、実装が軽い |
| 高 | 5. 自然言語一括追加 | UX 向上効果が大きい |
| 中 | 3. 献立プランナー | 既存レシピ機能の自然な拡張 |
| 中 | 1. スマート買い物提案 | 履歴蓄積が必要だが価値が高い |
| 中 | 6. 賞味期限トラッカー | 実用性が高い |
| 低 | 4. 予算管理 | 価格データの精度に課題 |
| 低 | 8. 季節提案 | 外部 API 依存 |
| 低 | 7. ルート最適化 | 店舗データの構築が重い |
| 低 | 9. 家族共有 | 認証基盤が必要 |
| 低 | 10. 生活リズム分析 | 長期データ蓄積が前提 |
