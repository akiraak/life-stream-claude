# 買い物リスト機能 仕様書

## 概要

日常の買い物管理を支援するシンプルなリスト機能。
Web ブラウザおよび iOS アプリから利用可能。

## 機能一覧

### 買い物食材管理

| 機能 | 説明 |
|------|------|
| 食材追加 | 名前を入力してリストに追加 |
| 食材編集 | 名前・カテゴリ・関連する料理を変更 |
| ~~食材削除~~ | ~~個別に削除~~ （削除ボタンは廃止、チェックで消す運用） |
| チェック | 購入済み食材をチェック |
| チェック済み一括削除 | チェック済み食材をまとめて削除 |
| 一覧表示 | 料理ごとにグループ化して表示（未関連食材は別枠） |

### 料理管理

| 機能 | 説明 |
|------|------|
| 料理追加 | 料理名を入力して登録 |
| 料理削除 | 料理を削除（関連する食材の紐付けも解除） |
| 食材紐付け | 買い物食材を料理に関連付ける |
| 紐付け解除 | 料理と食材の関連を解除 |

### 購入頻度サジェスト

| 機能 | 説明 |
|------|------|
| 食材サジェスト | 食材追加時に過去の購入頻度が高いものを候補表示（リスト内の既存食材は除外） |
| 料理サジェスト | 料理追加時に過去に追加した料理を候補表示（リスト内の既存料理は除外） |
| 初期表示 | 未入力状態で Top 3 を表示、入力に応じて絞り込み |

### AI 具材・レシピ提案

| 機能 | 説明 |
|------|------|
| 具材自動検索 | 料理追加時に AI（Gemini）がバックグラウンドで具材を検索。完了まで料理名横にスピナー表示 |
| 具材選択追加 | トグルチップ形式で具材を表示、タップで選択/解除し一括追加 |
| レシピ表示 | 同じモーダルに3件のレシピ（タイトル・概要・折りたたみ手順）を表示 |
| AI情報永続化 | 具材・レシピをDBに保存、同名料理の再追加時は前回のデータを引き継ぎ |
| 重複防止 | 同じ料理内の同名食材追加を防止（別料理・未分類は許可） |
| キャッシュアイコン | AI情報取得済みの料理に🧾アイコンを表示、タップで再表示 |

**フロー:**
1. ユーザーが料理名を入力して追加 → モーダル即閉じ
2. バックグラウンドで AI が具材+レシピを取得（料理名横にスピナー）
3. 完了後、具材チップ+レシピカード付きモーダルが表示される
4. ユーザーが具材チップをタップして選択 → 「追加」で買い物リストに一括追加
5. AI情報はDBに保存され、同名料理の再追加時はGemini不要で即表示
6. 料理名タップでいつでもキャッシュ済み情報を再表示可能

## API エンドポイント

### 買い物食材

#### `GET /api/shopping`
全食材一覧を取得する（未チェック優先でソート）。

#### `POST /api/shopping`
新しい食材を追加する。

```json
{ "name": "牛乳" }
```

#### `PUT /api/shopping/:id`
食材の状態を更新する（名前・カテゴリ・チェック状態）。

#### `DELETE /api/shopping/:id`
食材を削除する。

#### `DELETE /api/shopping/checked`
チェック済み食材を一括削除する。

### 料理

#### `GET /api/dishes`
全料理の一覧を取得する（紐付け食材含む）。

#### `POST /api/dishes`
新しい料理を追加する。

```json
{ "name": "カレーライス" }
```

#### `DELETE /api/dishes/:id`
料理を削除する（紐付けも連動削除）。

#### `POST /api/dishes/:id/items`
食材を料理に紐付ける。

```json
{ "itemId": 1 }
```

#### `DELETE /api/dishes/:id/items/:itemId`
料理と食材の紐付けを解除する。

### サジェスト

#### `GET /api/shopping/suggestions?q=`
購入頻度に基づく食材名のサジェストを返す。リスト内の未チェック食材は除外される。
空クエリで Top 3、テキスト入力時は Top 10。

**レスポンス例**:
```json
{ "success": true, "data": [{ "name": "牛乳", "count": 5 }] }
```

#### `GET /api/dishes/suggestions?q=`
過去に追加した料理名のサジェストを返す。現在リストにある料理は除外される。
空クエリで Top 3、テキスト入力時は Top 10。

### AI 機能

#### `POST /api/ai/suggest`
料理名をもとに AI（Gemini）が具材リスト+レシピを生成する（ステートレス）。
未ログインでも利用可能（`X-Device-Id` ヘッダ必須、JST 日次 3 回まで）。ログイン時は 20 回/日。
レスポンスヘッダ `X-AI-Remaining` に当日残回数を返す。

**リクエスト**:
```json
{ "dishName": "カレーライス", "extraIngredients": ["じゃがいも"] }
```

**レスポンス例**:
```json
{
  "success": true,
  "data": {
    "ingredients": [
      { "name": "じゃがいも", "category": "野菜" },
      { "name": "にんじん", "category": "野菜" }
    ],
    "recipes": [
      {
        "title": "基本のカレー",
        "summary": "玉ねぎをじっくり炒めて作る定番カレー",
        "steps": ["野菜を切る", "肉を炒める", "水を加えて煮込む"],
        "ingredients": [...]
      }
    ]
  }
}
```

#### `PUT /api/dishes/:id/ai-cache`
サーバモード時、AI 結果を料理行にキャッシュ保存する。body: `{ ingredients, recipes }`。

#### 廃止 API
- `GET /api/recipes/recommend` — 未使用のため廃止（`/api/ai/suggest` に統合）
- `POST /api/dishes/:id/suggest-ingredients` — 廃止（`/api/ai/suggest` + `/api/dishes/:id/ai-cache` に分離）
- `POST /api/claude` — 廃止（未使用）

## レスポンス形式

全 API 共通のレスポンス形式:

```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```
