# 買い物リスト機能 仕様書

## 概要

日常の買い物管理を支援するシンプルなリスト機能。
Web ブラウザおよび iOS アプリから利用可能。

## 機能一覧

### 買い物アイテム管理

| 機能 | 説明 |
|------|------|
| アイテム追加 | 名前を入力してリストに追加 |
| アイテム編集 | 名前・カテゴリ・関連する料理を変更 |
| ~~アイテム削除~~ | ~~個別に削除~~ （削除ボタンは廃止、チェックで消す運用） |
| チェック | 購入済みアイテムをチェック |
| チェック済み一括削除 | チェック済みアイテムをまとめて削除 |
| 一覧表示 | 料理ごとにグループ化して表示（未関連アイテムは別枠） |

### 料理管理

| 機能 | 説明 |
|------|------|
| 料理追加 | 料理名を入力して登録 |
| 料理削除 | 料理を削除（関連するアイテムの紐付けも解除） |
| アイテム紐付け | 買い物アイテムを料理に関連付ける |
| 紐付け解除 | 料理とアイテムの関連を解除 |

### 購入頻度サジェスト

| 機能 | 説明 |
|------|------|
| アイテムサジェスト | アイテム追加時に過去の購入頻度が高いものを候補表示（リスト内の既存アイテムは除外） |
| 料理サジェスト | 料理追加時に過去に追加した料理を候補表示（リスト内の既存料理は除外） |
| 初期表示 | 未入力状態で Top 3 を表示、入力に応じて絞り込み |

### AI 具材・レシピ提案

| 機能 | 説明 |
|------|------|
| 具材自動検索 | 料理追加時に AI（Gemini）がバックグラウンドで具材を検索。完了まで料理名横にスピナー表示 |
| 具材選択追加 | トグルチップ形式で具材を表示、タップで選択/解除し一括追加 |
| レシピ表示 | 同じモーダルに3件のレシピ（タイトル・概要・折りたたみ手順）を表示 |
| AI情報永続化 | 具材・レシピをDBに保存、同名料理の再追加時は前回のデータを引き継ぎ |
| 重複防止 | 同じ料理内の同名アイテム追加を防止（別料理・未分類は許可） |
| キャッシュアイコン | AI情報取得済みの料理に🧾アイコンを表示、タップで再表示 |

**フロー:**
1. ユーザーが料理名を入力して追加 → モーダル即閉じ
2. バックグラウンドで AI が具材+レシピを取得（料理名横にスピナー）
3. 完了後、具材チップ+レシピカード付きモーダルが表示される
4. ユーザーが具材チップをタップして選択 → 「追加」で買い物リストに一括追加
5. AI情報はDBに保存され、同名料理の再追加時はGemini不要で即表示
6. 料理名タップでいつでもキャッシュ済み情報を再表示可能

## API エンドポイント

### 買い物アイテム

#### `GET /api/shopping`
全アイテム一覧を取得する（未チェック優先でソート）。

#### `POST /api/shopping`
新しいアイテムを追加する。

```json
{ "name": "牛乳" }
```

#### `PUT /api/shopping/:id`
アイテムの状態を更新する（名前・カテゴリ・チェック状態）。

#### `DELETE /api/shopping/:id`
アイテムを削除する。

#### `DELETE /api/shopping/checked`
チェック済みアイテムを一括削除する。

### 料理

#### `GET /api/dishes`
全料理の一覧を取得する（紐付けアイテム含む）。

#### `POST /api/dishes`
新しい料理を追加する。

```json
{ "name": "カレーライス" }
```

#### `DELETE /api/dishes/:id`
料理を削除する（紐付けも連動削除）。

#### `POST /api/dishes/:id/items`
アイテムを料理に紐付ける。

```json
{ "itemId": 1 }
```

#### `DELETE /api/dishes/:id/items/:itemId`
料理とアイテムの紐付けを解除する。

### サジェスト

#### `GET /api/shopping/suggestions?q=`
購入頻度に基づくアイテム名のサジェストを返す。リスト内の未チェックアイテムは除外される。
空クエリで Top 3、テキスト入力時は Top 10。

**レスポンス例**:
```json
{ "success": true, "data": [{ "name": "牛乳", "count": 5 }] }
```

#### `GET /api/dishes/suggestions?q=`
過去に追加した料理名のサジェストを返す。現在リストにある料理は除外される。
空クエリで Top 3、テキスト入力時は Top 10。

### AI 機能

#### `GET /api/recipes/recommend`
未チェックの買い物アイテムをもとに AI がレシピを推薦する。

#### `POST /api/dishes/:id/suggest-ingredients`
料理名をもとに AI（Gemini）が具材リスト+レシピを生成する。
DBにキャッシュがあればGemini呼び出しをスキップし即返却。

**リクエスト**: なし（料理名は `:id` から取得）

**レスポンス例**:
```json
{
  "success": true,
  "data": {
    "dishId": 1,
    "dishName": "カレーライス",
    "ingredients": [
      { "name": "じゃがいも", "category": "野菜" },
      { "name": "にんじん", "category": "野菜" }
    ],
    "recipes": [
      {
        "title": "基本のカレー",
        "summary": "玉ねぎをじっくり炒めて作る定番カレー",
        "steps": ["野菜を切る", "肉を炒める", "水を加えて煮込む"]
      }
    ]
  }
}
```

## レスポンス形式

全 API 共通のレスポンス形式:

```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```
